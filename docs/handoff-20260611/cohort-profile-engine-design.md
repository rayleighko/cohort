## 1. 전체 구조 — 2채널 × 2단계

```
[채널 1: 자기보고]                    [채널 2: 행동 관찰]
 라이트(30초, 2문항)                   behavioral_event 테이블 (기존)
   ↓ 업그레이드 유도                    - 대시보드 확인 빈도
 정식(5분, 13+8문항)                   - 트리거 반응 속도/무시율
   ↓                                  - (Phase 3+) 실거래: 회전율·처분효과
   └──────────→ ProfileEngine ←───────┘
                    ↓
        성향 스냅샷 (90일 유효, 재검사 유도)
```

행동 채널이 자기보고를 보정한다는 게 차별점: "설문에선 안정형이라더니 행동은 공격형" 같은 괴리 자체가 가장 가치 있는 인사이트이자, MBTI류 서비스가 못 하는 것.

## 2. 설문 설계

### 2-1. 라이트 (비로그인 허용, 30초)
- Q1: SCF 단일 문항 (4지선다 — 상당한 위험 감수 / 평균 이상 / 평균 / 위험 회피)
- Q2: 능동/수동 부 축적 경로 1문항 (Barnewall: 직접 일군 자산인가, 급여·저축 중심인가)
- 출력: 임시 밴드 + "정식 진단으로 당신의 유형을 확인하세요" CTA → 가입 전환 포인트

### 2-2. 정식 (로그인, ~5분, 21문항)
**파트 A — 위험감수성 (GL-RTS 13문항 번안)**
- Grable & Lytton(1999) 원척도 채점표 그대로(가중치 포함), 합산 점수.
- 정직성 표기(결과 화면 하단): "Grable-Lytton(1999) 척도의 한국어 번안으로, 번안판 자체의 심리측정 타당화는 거치지 않았습니다."
- 한국 맥락 어휘만 현지화(401k→연금저축 등), 문항 구조·선택지 수는 원척도 유지 — 채점표 호환 보존.

**파트 B — 편향 성향 스크리닝 (8문항, 시나리오형)**
Pompian BIT의 감정형(emotional) vs 인지형(cognitive) 축 판별용. 각 편향 2문항:
- 손실회피/후회회피 (감정형): "-15% 종목, 어떻게 하시겠어요?" 류 시나리오
- 현상유지 (감정형)
- 군집/최신편향 (인지형): "커뮤니티에서 모두가 사는 종목을 보면?"
- 과신/통제착각 (감정형-Accumulator형): "본인의 종목 선택 능력은 평균 대비?"
- ⚠️ 이 8문항은 자체 제작(검증된 단일 척도가 없음) — 문항별로 어떤 편향 문헌에 기반했는지 주석으로 남기고, "Pompian(2012) 프레임워크 기반 자체 문항"으로 표기.

### 2-3. 채점 → 유형 (Pompian BIT 그리드)
Pompian 원전의 구조 그대로: 위험감수성 저→고 축 위에 4유형이 놓이고, 양 끝(Preserver/Accumulator)은 감정편향 주도, 중간(Follower/Individualist)은 인지편향 주도.

```
GL-RTS 점수 ──→ 저 ──────── 중하 ──────── 중상 ──────── 고
                 │            │             │            │
편향 축:      감정형        인지형         인지형        감정형
유형:      Passive      Friendly     Independent    Active
           Preserver    Follower     Individualist  Accumulator
```

- 1차 분류: GL-RTS 4분위 밴드
- 2차 보정: 파트 B 점수가 밴드 경계에서 감정/인지 축으로 타이브레이크
- 동시에 금투협 5단계(안정형~공격투자형) 점수 별도 저장 — 향후 제휴/라이선스 자산
- CFA 커리큘럼의 경고("유형은 혼재·변동") 반영: 결과에 주유형 + 부유형 확률 표시, 단정 라벨 금지

## 3. 결과 화면 / 공유 카드

- 4유형 × Aurora(🕊 차분)/Vesper(🦅 기민) 브랜드 결합 네이밍 (예시 — 네이밍은 같이 정할 것):
  - Preserver = "등대형" / Follower = "물결형" / Individualist = "나침반형" / Accumulator = "맹금형"
- 결과 구성: ①유형 카드(공유용 OG 이미지, @vercel/og) ②내 위험감수성 점수와 분포 위치 ③**이 유형이 빠지기 쉬운 편향 Top 3 + 실제 논문 수치**("가장 활발히 거래한 집단은 시장 대비 연 −6.5%p — Barber & Odean 2000") ④맞춤 개입 제안(다음 단계 = IPS 위저드 CTA)
- **금지**: 종목·섹터·상품 추천. 허용: 자산군 *교육 콘텐츠* 링크(개별성 없는 일반 정보 — 유사투자자문 경계 내)
- 공유 카드에는 유형+한 줄 설명만(점수·재무정보 미포함 — 프라이버시)
- "역게임화" 브랜드와의 정합: 공유 카드는 거래를 부추기지 않고 자기이해를 공유하는 것이므로 충돌 없음. 단 카피에서 수익 암시 금지.

## 4. 데이터 모델 (기존 스키마 확장)

```sql
-- user_investment_profile 확장 (기존 0007 마이그레이션 위에)
alter table user_investment_profile add column
  gl_rts_score smallint,            -- 원척도 합산
  gl_rts_answers jsonb,             -- 문항별 응답 (재채점 가능하게)
  bias_answers jsonb,
  bit_type text check (bit_type in ('preserver','follower','individualist','accumulator')),
  bit_secondary text,               -- 부유형
  bias_orientation text check (bias_orientation in ('emotional','cognitive')),
  kofia_band smallint,              -- 1=안정형 ~ 5=공격투자형
  profile_version text not null default 'v1',  -- 채점 로직 버전 (재채점 추적)
  assessed_at timestamptz,
  retest_due_at timestamptz;        -- assessed_at + 90일

-- 이력 보존 (성향 변화 추적 = 장기 가치)
create table profile_snapshot ( ... 위 필드의 시점 사본, RLS: 본인만 );
```

행동 채널 보정치는 `behavioral_event`에서 주기 배치로 계산해 snapshot에 `behavior_adjustment jsonb`로 병기 — 자기보고 점수를 덮어쓰지 않고 나란히 보여줌(정직성).

## 5. 구현 — 수직 슬라이스 + TDD 계획

핵심 로직(Ray가 직접 타이핑, 테스트 먼저):
1. `scoreGlRts(answers): number` — 순수 함수. Red: 원척도 채점표 기반 케이스 + 경계값 + 불완전 응답 거부
2. `classifyBit(glScore, biasScores): {primary, secondary, orientation}` — 순수 함수. Red: 4분위 경계에서 타이브레이크 케이스, 동점 처리

보일러플레이트(에이전트에 위임 가능): 설문 UI 스텝퍼, zod 스키마, OG 이미지 라우트, 마이그레이션 골격.

API: `POST /api/profile/assessment` (채점은 서버에서만 — 클라이언트 신뢰 금지), `GET /api/profile`. 기존 `/api/survey`와의 관계는 통합 vs 병행 결정 필요.

## 6. ADR 초안 — "0006: 투자 성향 진단 체계 선정"

- **Context**: 성향 진단이 제품 코어. 자체 설문은 빠르나 방어 불가, 검증 척도는 신뢰 가능하나 한국어 타당화 부재.
- **Options**: (a) 자체 15문항(현행 onboarding 확장) (b) GL-RTS 13문항 번안 + BIT (c) DOSPERT 투자 도메인 (d) 외부 진단 API
- **Decision**: (b). 무료 공개·채점표 존재·재현 가능, BIT가 유형→편향→개입 파이프라인 제공, 금투협 5단계 호환 매핑 가능.
- **Trade-offs**: 번안 타당화 비용은 미지불(표기로 정직성 유지), 편향 8문항은 자체 제작이라 약한 고리, MBTI식 강한 단정 마케팅 불가(확률 표시).
- **Consequences**: profile_version으로 채점 로직 진화 추적, 향후 응답 데이터 쌓이면 자체 신뢰도(Cronbach α) 계산해 공개 — 이것 자체가 포트폴리오 숫자.

## 8. 비로그인 → 재방문 퍼널 (확정 설계)

```
랜딩 히어로: "내 투자 성향, 2문항으로 30초 만에"
  → 라이트 2문항 (비로그인, LLM 미사용·순수 채점이라 비용 ~0)
  → 즉시 부분 결과: 임시 밴드 + 가능 유형 2개를 실루엣으로 (호기심 갭)
  → CTA: "정식 진단으로 내 유형 확정하기" → 가입
  → 통합 설문(GL-RTS 13 + 기존 사실문항) → 유형 카드 + OG 공유 이미지
  → 재방문 루프: ① 90일 재검사 푸시(기존 web-push 재활용)
                ② 스냅샷 타임라인("내 성향 변화") — §9
                ③ 기존 mascot streak·트리거 알림과 연결
```

- 어뷰징 대응(= 봇/반복 호출로 인한 DB 쓰기 스팸 방지): 익명 결과는 **DB에 저장하지 않음** — 결과를 서명 토큰(JWT)으로 클라이언트에 주고, 가입 시 클레임하여 그때 저장. + IP 레이트리밋. LLM을 안 쓰므로 비용 리스크는 낮음.
- 공유 카드는 유형+한 줄 설명만(점수·재무정보 제외).

## 9. 성향 스냅샷 타임라인 (Ray 아이디어 반영 — 장기 코어 기능)

`profile_snapshot`(§4)이 이 기능의 기반. 확장 설계:
- 모든 정식 설문 제출 = 스냅샷 1건 (제출 시각, 채점 버전, 유형, 점수 전부 보존)
- 타임라인 뷰: "2026-06 물결 → 2026-09 나침반" + 그 사이의 행동 데이터(회전율·트리거 반응) 병기
- Phase 2+: 스냅샷에 당시의 **투자 원칙(IPS) 버전**을 연결 → Phase 4: 각 원칙 버전의 **백테스트 결과·실성과**를 스냅샷에 연결 → "성향이 이렇게 변했고, 그때 원칙은 이랬으며, 결과는 이랬다"는 개인 투자 연대기
- 스키마 함의: `profile_snapshot.id`를 IPS 문서·백테스트 런이 FK로 참조 (지금은 컬럼만 예약: `linked_principle_id nullable`)
